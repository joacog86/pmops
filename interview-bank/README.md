Cada historia = una sesión de `/interview-prep` completada.

Cuando llegues a una entrevista, revisá este banco antes. No para memorizar — para recordar lo que ya hiciste.

## Estructura de carpetas

```
interview-bank/
├── product-sense/     ← Diseño y mejora de productos
├── execution-metrics/ ← Métricas, análisis, investigación
├── strategy/          ← Decisiones estratégicas, entrada a mercados
└── behavioral/        ← STAR stories de situaciones reales
```

## Cómo usar este banco en una entrevista real

1. **Antes de la entrevista** — Leé los últimos 2-3 stories del tipo que más probablemente te pregunten
2. **Durante** — No recitás la historia. Usás los elementos reales (el usuario que nombraste, el trade-off que identificaste, la métrica que elegiste)
3. **La confianza viene de** haber pensado en serio sobre el producto, no de haber ensayado la respuesta
